# 2. Setup

The widgetbook package offers a Flutter Widget called `Widgetbook`, which enables you to inject custom widgets from your app. Before we start, let's take a look at a folder structure in Flutter projects.

```dart
your_flutter_project/
├─ android/
├─ ios/
├─ lib/
│ ├─ main.dart
│ ├─ src/
│ │ ├─ widgets/
│ │ ├─ models/
│ │ └─ services/
│ └─ ...
├─ assets/
│ ├─ images/
│ ├─ fonts/
│ └─ ...
├─ test/
├─ pubspec.yaml
└─ ...
```

To initialize Widgetbook in your Flutter project, follow the steps below:

  <TabItem value="generator">

1- Create a new file in your `lib` folder called `ui_catalogs.dart`. Alternatively, you can annotate an existing code element using the `@WidgetbookApp` annotation.

```dart
your_flutter_project/
│─ ...
├─ lib/
│ ├─ main.dart
│ ├─ ui_catalogs.dart
│ ├─ src/
├─ pubspec.yaml
└─ ...
```

<Info>
  The `widgetbook_generator` package will create a corresponding `.widgetbook.dart` file based on
  the location of the `@WidgetbookApp` annotation. The annotated code element does not affect the
  generated output code.
</Info>

2- Add the following code snippet to your `ui_catalogs.dart` file:

    ```dart title="ui_catalogs.dart"
    // Setup of Widgetbook: ui_catalogs.dart
    import 'package:widgetbook_annotation/widgetbook_annotation.dart';

    @WidgetbookApp.material()
    late int notUsed;
    ```

    3- Execute the `build_runner` by running the following command:
    ```bash
    flutter pub run build_runner build
    ```

<Info>You may rename `ui_catalogs.dart` to your preferred name that fits your project.</Info>

The command generates the `ui_catalogs.widgetbook.dart` file.

```dart
your_flutter_project/
│─ ...
├─ lib/
│ ├─ main.dart
│ ├─ ui_catalogs.dart
│ ├─ ui_catalogs.widgetbook.dart
│ ├─ src/
├─ pubspec.yaml
└─ ...
```

    4- The generated app.widgetbook.dart file should look like this:

    ```dart title="app.widgetbook.dart"
        // Generated Widgetbook file: app.widgetbook.dart
        import 'package:flutter/material.dart';
        import 'package:widgetbook/widgetbook.dart';

        void main() {
        runApp(HotReload());
        }

        class HotReload extends StatelessWidget {
        const HotReload({Key? key}) : super(key: key);

        @override
        Widget build(BuildContext context) {
            return Widgetbook.material(
            addons: [],
            directories: [],
            );
        }
    }
    ```

With these steps, you have successfully set up Widgetbook using the generator approach with annotations.

## Custom Theme

`Widgetbook` allows developers to define whatever `Theme` they have defined for their app. To accompany every theme and the ones defined by Flutter, the following constructors exist:

| Theme        | Constructor             | Defaults to                      |
| ------------ | ----------------------- | -------------------------------- |
| Custom Theme | `Widgetbook<YourTheme>` | ➖                               |
| Material     | `Widgetbook.material`   | `Widgetbook<ThemeData>`          |
| Cupertino    | `Widgetbook.cupertino`  | `Widgetbook<CupertinoThemeData>` |

Check out [Custom Theme](#) docs to learn more.
